[center][size=5][b]Calamity - Reactive Loot & Affixes[/b][/size]
Player-centric ARPG-style instance affix mod for Skyrim SE/AE[/center]

[hr]

[size=4][b]한국어[/b][/size]

[b]모드 소개[/b]
Calamity - Reactive Loot & Affixes는 Skyrim SE/AE용 플레이어 중심 ARPG 스타일 어픽스 모드입니다.
아이템 인스턴스(ExtraUniqueID) 단위로 상태를 추적하며, 현재 빌드 기본 정책은 [b]재련(Reforge) 중심[/b]입니다.

[b]현재 빌드 핵심 정책[/b]
[list]
[*]아이템 획득/제작 시 [b]자동 어픽스 부여 없음[/b]
[*][b]재련 오브(Reforge Orb)[/b] 사용 시 선택 장비에 어픽스를 부여/재롤
[*]슬롯 모델: [b]룬워드 1 + 일반 어픽스 최대 3[/b]
[*]이름 마커(★ 계열) + Prisma 툴팁으로 인스턴스 상태 확인
[/list]

[b]룬워드[/b]
[list]
[*]Diablo 2 스타일 [b]94개 레시피[/b]
[*]룬워드 조각 수집 -> 레시피 완성 -> 장비 적용
[*]완성 룬워드 장비도 재련 가능 (룬워드 + 일반 어픽스 동시 재롤)
[*]효과 구성: [b]직접 정의형 16개 + 자동 합성형 78개[/b]
[*]Adaptive 계열은 자동 선택 기본 유지, [code]modeCycle[/code] 효과는 [b]수동 오버라이드 모드[/b] 지원
[*]룬워드 패널에서 선택 레시피의 [b]효과/권장 베이스/룬 순서/상세[/b]를 상태 영역에서 확인 가능
[/list]

[b]드랍 정책 (현재)[/b]
[list]
[*]드랍 모드: [b][color=#66ccff]hybrid 고정[/color][/b]
[*]시체: [b]SPID DeathItem 분배 우선[/b]
[*]컨테이너: [b]SKSE 런타임 롤[/b]
[*]월드(바닥) 픽업: [b]드랍 없음[/b]
[*]기본 확률: 룬워드 조각 [b]8%[/b], 재련 오브 [b]5%[/b]
[*]MCM에서 확률 조정 가능 (런타임 즉시 반영)
[/list]

[b]전투 시스템[/b]
[list]
[*]Proc 발동 + ICD(내부 쿨다운) + 중복 히트 방지
[*]Best Slot Wins: 1어픽스=100%, 2어픽스=80%, 3어픽스=65%
[*]크리티컬 히트 추가 증폭(SKSE 훅 기반)
[/list]

[b]UI & 설정[/b]
[list]
[*]Prisma UI 기반 툴팁/조작 패널
[*]MCM 설정 패널(확률, 핫키, 언어, 런타임 옵션)
[*]한국어/영어 전환 지원
[*][code]user_settings.json[/code] 기반 설정 영속화
[/list]

[b]적용 범위 (중요)[/b]
[list]

[size=4][b]English[/b][/size]

[b]Overview[/b]
Calamity - Reactive Loot & Affixes is a player-centric ARPG-style affix mod for Skyrim SE/AE.
It tracks item instances via ExtraUniqueID, and the current build is [b]Reforge-centric[/b].

[b]Current Core Policy[/b]
[list]
[*][b]No automatic affix assignment[/b] on loot/craft
[*]Use [b]Reforge Orb[/b] to grant/reroll affixes on selected gear
[*]Slot model: [b]1 runeword + up to 3 regular affixes[/b]
[*]Star markers (★ series) + Prisma tooltip for instance readability
[/list]

[b]Runewords[/b]
[list]
[*][b]94 Diablo 2-style recipes[/b]
[*]Collect runeword fragments -> complete recipe -> apply to equipment
[*]Completed runeword gear can also be reforged (runeword + regular affixes reroll together)
[*]Effect composition: [b]16 hand-authored runewords + 78 runtime-synthesized runewords[/b]
[*]Adaptive effects keep auto element selection by default, and [code]modeCycle[/code] entries support [b]manual override mode[/b]
[*]The runeword panel now shows selected recipe [b]effect/base/rune-order/detail[/b] directly in the status area
[/list]

[b]Drop Policy (Current)[/b]
[list]
[*]Drop mode: [b][color=#66ccff]hybrid only[/color][/b]
[*]Corpses: [b]SPID DeathItem distribution first[/b]
[*]Containers: [b]SKSE runtime roll[/b]
[*]World pickups: [b]no drops[/b]
[*]Default rates: runeword fragment [b]8%[/b], reforge orb [b]5%[/b]
[*]Rates are adjustable via MCM (applies immediately at runtime)
[/list]

[b]Combat System[/b]
[list]
[*]Proc chance + ICD (internal cooldown) + duplicate-hit protection
[*]Best Slot Wins: 1 affix=100%, 2 affixes=80%, 3 affixes=65%
[*]Critical hit amplification via SKSE hook
[/list]

[b]UI & Settings[/b]
[list]
[*]Prisma UI tooltip/control panel
[*]MCM settings (rates, hotkeys, language, runtime options)
[*]Korean / English language switch
[*]Persistent settings via [code]user_settings.json[/code]
[/list]

[b]Scope (Important)[/b]
[list]
[*]Based on player inventory/equipment
[*]Combat triggers support player + player-owned summons/proxies
[*]Independent affix ownership/runtime for regular followers/NPC equipment is not supported
[/list]

[b]Required Dependencies[/b]
[list]
[*]SKSE64
[*]Address Library for SKSE Plugins
[*]Prisma UI
[/list]

[b]Recommended / Optional[/b]
[list]
[*]SkyUI (recommended)
[*]KID (recommended)
[*]SPID (recommended)
[*]MCM Helper (recommended)
[*]I4 (optional)
[/list]

[b]Installation (MO2 recommended)[/b]
[list=1]
[*]Download the Main File
[*]Install with MO2
[*]Enable [code]CalamityAffixes.esp[/code]
[*]Launch via SKSE
[/list]

[b]Notes[/b]
[list]
[*]Without Prisma UI, tooltip/control panel UI will not be shown.
[*]Overly broad KID DoT tagging may cause side effects.
[*]Leveled-list (LVLI) injection/override is not used.
[/list]
