[center][size=5][b]Calamity - Reactive Loot & Affixes[/b][/size]
Player-centric ARPG-style instance affix mod for Skyrim SE/AE[/center]

[hr]

[size=4][b]한국어[/b][/size]

[b]모드 소개[/b]
Calamity - Reactive Loot & Affixes는 Skyrim SE/AE용 플레이어 중심 ARPG 스타일 어픽스 모드입니다.
아이템 인스턴스(ExtraUniqueID) 단위로 상태를 추적하며, 현재 빌드 기본 정책은 [b]재련(Reforge) 중심[/b]입니다.

[b]현재 빌드 핵심 정책[/b]
[list]
[*]아이템 획득/제작 시 [b]자동 어픽스 부여 없음[/b]
[*][b]재련 오브(Reforge Orb)[/b] 사용 시 선택 장비에 어픽스를 부여/재롤
[*]슬롯 모델: [b]룬워드 1 + 일반 어픽스 최대 3[/b]
[*]이름 마커(★ 계열) + Prisma 툴팁으로 인스턴스 상태 확인
[/list]

[b]룬워드[/b]
[list]
[*]Diablo 2 스타일 [b]94개 레시피[/b]
[*]룬워드 조각 수집 -> 레시피 완성 -> 장비 적용
[*]완성 룬워드 장비 재련 시 룬워드는 보존되고 일반 어픽스만 재롤
[*]룬워드 재변환 시 기존 룬워드를 새 룬워드로 교체 가능
[*]효과 구성: [b]94개 전부 JSON 개별 정의[/b]
[*]Adaptive 계열은 자동 선택 기본 유지, [code]modeCycle[/code] 효과는 [b]수동 오버라이드 모드[/b] 지원
[*]룬워드 패널에서 [b]효과/권장 베이스/룬 순서/상세[/b]를 한국어/영어 설정에 맞춰 선택 상태·검색·hover에서 확인 가능
[/list]

[b]드랍 정책 (현재)[/b]
[list]
[*][code]hybrid[/code]는 구버전 호환 토큰이며, 실제 권한은 [b]SKSE death-event eligible hostile corpse-only[/b]
[*]플레이어 또는 player-owned summon/proxy가 처치한 적대 시체 인벤토리에 성공한 통화를 조용히 직접 추가
[*]팔로워/동료, 소환·지휘 액터, 아동, player-owned/비적대 피해자와 환경 오브젝트·player-owned가 아닌 독립 NPC/팔로워 처치는 제외
[*][b]일반 상자/컨테이너 활성화, 픽업, 월드 생성, 새 SPID 통화 분배 없음[/b]
[*]일반 적 기본 확률: 룬워드 조각 [b]12%[/b], 재련 오브 [b]7%[/b] (MCM 변경은 다음 적격 일반 적 사망부터 반영)
[*][code]Unique[/code] 고유·네임드 적: 룬워드 조각 [b]70%[/b] / 재련 오브 [b]30%[/b] 중 1개 확정
[*][code]LocRefTypeBoss[/code] 보스: 룬워드 조각 1개 + 재련 오브 1개 확정 ([code]Boss[/code]가 [code]Unique[/code]보다 우선)
[*]고유·보스 확정 보상은 일반 확률 판정과 피티를 소비하거나 초기화하지 않음
[*]룬 가중치: [code]El-Amn=4 / Sol-Um=3 / Mal-Lo=2 / Sur-Zod=1[/code] (최대 4:1)
[*]룬 조각 99회 연속 실패 피티 유지; 피티와 시체별 중복 방지 ledger는 [code]CCRT[/code] 코세이브 레코드에 저장
[*]업데이트 전 SPID 통화가 남은 전환 시체는 룬 조각/재련 오브 카테고리별 중복 판정 생략
[*]SPID는 필요하지 않으며 동봉 [code]CalamityAffixes_DISTR.ini[/code]는 빈 호환 산출물
[/list]

[size=4][b]English[/b][/size]

[b]Overview[/b]
Calamity - Reactive Loot & Affixes is a player-centric ARPG-style affix mod for Skyrim SE/AE.
It tracks item instances via ExtraUniqueID, and the current build is [b]Reforge-centric[/b].

[b]Current Core Policy[/b]
[list]
[*][b]No automatic affix assignment[/b] on loot/craft
[*]Use [b]Reforge Orb[/b] to grant/reroll affixes on selected gear
[*]Slot model: [b]1 runeword + up to 3 regular affixes[/b]
[*]Star markers (★ series) + Prisma tooltip for instance readability
[/list]

[b]Runewords[/b]
[list]
[*][b]94 Diablo 2-style recipes[/b]
[*]Collect runeword fragments -> complete recipe -> apply to equipment
[*]Reforging completed runeword gear preserves the runeword and rerolls only regular affixes
[*]Runeword re-transmutation can replace an existing runeword with a new one
[*]Effect composition: [b]all 94 runewords individually defined in JSON[/b]
[*]Adaptive effects keep auto element selection by default, and [code]modeCycle[/code] entries support [b]manual override mode[/b]
[*]The panel shows bilingual [b]effect/base/rune-order/detail[/b] consistently in selection, search, and hover states
[/list]

[b]Drop Policy (Current)[/b]
[list]
[*][code]hybrid[/code] is a legacy config token; actual authority is [b]SKSE death-event eligible-hostile-corpse-only[/b]
[*]Successful currency is silently inserted into a hostile corpse killed by the player or a player-owned summon/proxy
[*]Followers/teammates, summoned or commanded victims, children, player-owned/non-hostile victims, environmental-object kills, and kills by independent non-player-owned NPCs/followers are excluded
[*][b]No generic container activation, pickup roll, world spawn, or new SPID currency distribution[/b]
[*]Normal-enemy rates: runeword fragment [b]12%[/b], reforge orb [b]7%[/b] (MCM changes apply to the next eligible normal-enemy death)
[*]Unique/named actors: one guaranteed currency reward, selected as [b]70%[/b] fragment / [b]30%[/b] reforge orb
[*][code]LocRefTypeBoss[/code] actors: one guaranteed fragment plus one guaranteed reforge orb ([code]Boss[/code] overrides [code]Unique[/code])
[*]Unique/boss guarantees neither run additional normal rolls nor advance/reset normal pity
[*]Rune weights: [code]El-Amn=4 / Sol-Um=3 / Mal-Lo=2 / Sur-Zod=1[/code] (maximum 4:1)
[*]The 99-failure fragment pity remains; pity and the per-corpse duplicate ledger are saved in the [code]CCRT[/code] co-save record
[*]Transition corpses carrying old SPID currency skip duplicate rolls per fragment/orb category
[*]SPID is not required; bundled [code]CalamityAffixes_DISTR.ini[/code] is an empty compatibility artifact
[/list]

[b]Combat System[/b]
[list]
[*]Proc chance + ICD (internal cooldown) + duplicate-hit protection
[*]Best Slot Wins: 1 affix=100%, 2 affixes=80%, 3 affixes=65%
[*]Critical hit amplification via SKSE hook
[/list]

[b]UI & Settings[/b]
[list]
[*]Prisma UI tooltip/control panel
[*]MCM settings (rates, hotkeys, language, runtime options)
[*]Korean / English language switch
[*]Bilingual recipe effect/base/rune-order/detail in selection, search, and hover
[*]Viewport clamping + wheel/scrollbar/keyboard scrolling + ARIA accessibility
[*]Persistent settings via [code]user_settings.json[/code]
[/list]

[b]Scope (Important)[/b]
[list]
[*]Based on player inventory/equipment
[*]Combat triggers support player + player-owned summons/proxies
[*]Independent affix ownership/runtime for regular followers/NPC equipment is not supported
[/list]

[b]Required Dependencies[/b]
[list]
[*]SKSE64
[*]Address Library for SKSE Plugins
[*]Prisma UI
[/list]

[b]Recommended / Optional[/b]
[list]
[*]SkyUI (recommended)
[*]KID (recommended)
[*]MCM Helper (recommended)
[*]I4 (optional)
[/list]

[b]Installation (MO2 recommended)[/b]
[list=1]
[*]Download the Main File
[*]Install with MO2
[*]Enable [code]CalamityAffixes.esp[/code]
[*]Launch via SKSE
[/list]

[b]Notes[/b]
[list]
[*]Without Prisma UI, tooltip/control panel UI will not be shown.
[*]Overly broad KID DoT tagging may cause side effects.
[*]Leveled-list (LVLI) injection/override is not used.
[*]If a new version is enabled as a separate MO2 mod, conflict priority may let the older [code]CalamityAffixes_DISTR.ini[/code] override the new empty compatibility file. Replace/overwrite the old mod or disable the old DISTR file.
[*]Existing gear, currencies, completed runewords, and prior co-save records remain compatible; no new game is required.
[/list]
