# Calamity - Reactive Loot & Affixes v1.3.2

> Pre-release / Prisma 패널 성능·조작감 피드백용
> 날짜: 2026-07-13

## 요약

v1.3.2는 Prisma UI 프레임워크와 v1.3.1의 View 수명주기 복구를 그대로 유지하면서, 패널을 연 뒤 계속 느껴질 수 있던 주기적 끊김과 딱딱한 입력 반응을 줄이는 공개 테스트 패치입니다.

이번 버전은 패널이 열려 있는 동안 같은 SKSE task가 쌓이지 않도록 worker Tick을 병합하고, 거의 변하지 않는 룬워드 레시피 카탈로그를 반복 계산 경로에서 분리합니다. HTML 패널은 동일 항목 DOM을 재사용하고, 휠·상태 전환·크기 조절을 프레임 친화적인 방식으로 처리합니다.

어픽스·룬워드 효과와 룬 경제는 이번 성능 패치의 대상이 아닙니다.

## 메인 스레드 및 패널 데이터 경량화

- worker가 예약하거나 실행 중인 Prisma Tick을 최대 하나로 제한합니다. 이전 Tick이 아직 처리되지 않았다면 같은 Tick을 task queue에 추가하지 않습니다.
- 94개 룬워드의 이름, 룬 순서, 효과 설명과 권장 베이스처럼 거의 변하지 않는 정적 카탈로그를 주기 갱신에서 분리합니다.
- 선택한 베이스·레시피, 보유 파편, 삽입 진행도와 적용 가능 상태처럼 실제로 변하는 데이터만 필요한 시점에 갱신합니다.
- 이미 적용된 panel open 상태는 다시 보내지 않습니다.
- 직전과 내용이 같은 interop payload는 JavaScript에 중복 전달하지 않습니다.
- v1.3.1에서 추가한 Prisma API 탐지, View 유효성 확인, DOM-ready timeout과 재시도 경로는 유지합니다.

## 패널 조작감 개선

- 레시피 목록은 항목 key를 기준으로 기존 DOM을 재사용해 선택·검색 변경 때 전체 목록을 불필요하게 다시 만들지 않습니다.
- 레시피 검색용 텍스트는 항목 생성 시 캐시해, 검색 입력마다 효과 설명 전체를 반복 조합하는 비용을 줄입니다.
- 휠 입력은 최소 136.8px을 즉시 점프하던 방식 대신 delta와 경과 시간을 제한한 시간 기반 스크롤을 사용합니다.
- 큰 `box-shadow`를 무한 반복하던 강조 애니메이션을 제거하고, 짧은 transform·opacity 중심 전환으로 교체합니다.
- 상태 전환을 다시 시작하기 위해 `offsetWidth`를 동기 조회하던 강제 reflow를 제거합니다.
- 연속적인 화면 갱신은 `requestAnimationFrame` 단위로 합쳐 CEF가 한 프레임에 같은 레이아웃을 여러 번 계산하지 않도록 합니다.

## 호환성 및 변경하지 않은 항목

- 새 게임은 필요하지 않습니다.
- 기존 장비, 보유 파편, 재련 오브와 완성된 룬워드는 그대로 유지됩니다.
- 세이브 및 어픽스·룬워드 직렬화 형식은 변경하지 않았습니다.
- 어픽스 및 룬워드 효과 정의는 변경하지 않았습니다.
- 룬 가중치와 드롭 분포는 변경하지 않았습니다.
- 기본 파편 드롭률 `8%`는 변경하지 않았습니다.
- 99회 무드롭 피티는 변경하지 않았습니다.
- SKSE64, Address Library 및 Prisma UI 요구 사항은 기존과 같습니다.
- v1.3.1의 MCM Prisma 상태 확인, 룬워드 상태 표시와 세이브당 1회 안전 통화 복구를 유지합니다.

## 공개 테스트 피드백 요청

패널을 연 직후뿐 아니라 30초 이상 연 상태에서 테스트해 주세요. 가능하면 다음 형식으로 알려 주세요.

```text
해상도와 창 모드:
평균 게임 FPS / 패널을 연 뒤 FPS:
Prisma UI 버전:
패널을 30초 이상 연 뒤 지속 끊김 여부:
약 2초 간격의 짧은 spike 여부:
레시피 목록 휠 스크롤 체감: 너무 느림 / 적당함 / 너무 빠름
패널 드래그·크기 조절 체감:
검색 입력과 탭 전환 반응:
룬워드 레시피 수와 장착 장비 수:
CalamityAffixes.log의 Prisma telemetry 또는 timeout/invalid 반복 여부:
다른 UI 모드와의 조합:
```

특히 다음을 구분해 알려 주시면 원인 분석에 도움이 됩니다.

- 패널을 여는 순간에만 한 번 끊기는지
- 패널을 연 상태에서 약 300ms 간격으로 계속 끊기는지
- 패널을 연 상태에서 약 2초 간격의 짧은 끊김이 반복되는지
- 휠·검색·드래그·크기 조절 중에만 끊기는지
- 패널을 닫으면 즉시 정상 프레임으로 돌아오는지

## 알려진 제한

- 실제 프레임과 입력 지연은 해상도, CEF 렌더링, GPU 드라이버와 함께 설치한 UI 모드에 따라 달라질 수 있습니다.
- 이 패치는 불필요한 반복 작업과 비싼 화면 갱신을 줄이지만 Prisma UI 자체를 교체하지는 않습니다.
- MCM 폴백은 상태 확인과 안전 복구만 제공하며 아이템별 툴팁, 레시피 선택과 재련 조작에는 Prisma UI가 계속 필요합니다.
- 자동 테스트는 실제 Skyrim 세션의 CEF frame pacing과 전체 모드 조합을 완전히 재현하지 않습니다.
