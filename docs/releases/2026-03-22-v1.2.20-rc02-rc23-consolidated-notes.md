# v1.2.20 RC 시리즈 통합 노트 (rc02~rc23)

- 기준 범위: `v1.2.20-rc02` ~ `v1.2.20-rc23`
- 작성 기준일: 2026-03-22
- 비교 기준 정식 릴리즈: `v1.2.19.2` → `v1.2.20`
- 목적: `v1.2.20` 정식 릴리즈에 들어간 RC 변경 흐름을 유저가 한 문서에서 읽을 수 있게 정리

## 참고

- 저장소에는 `v1.2.20-rc01` 태그가 없습니다.
- 그래서 이 통합 노트는 저장소에서 확인 가능한 첫 태그인 `v1.2.20-rc02`부터 정리합니다.

## 한눈에 보기

- 룬워드 효과군을 대폭 재설계하고, 중복 효과를 고유 효과로 바꾸면서 빌드 개성을 강화했습니다.
- 프리픽스/서픽스 체계를 전면 손봐 파밍 결과와 설명 가독성을 정리했습니다.
- 룬워드 워크벤치와 Prisma 패널 UX를 여러 RC에 걸쳐 다듬어 조작 체감과 정보 노출을 개선했습니다.
- 피격 프리징, SE/AE 공용 빌드 문제, 경로 이동/패키징 불안정 같은 릴리즈 리스크를 후반 RC에서 정리했습니다.

## 유저 관점 핵심 변화

### 콘텐츠와 빌드 다양성

- 룬워드 효과 94개를 스카이림 마법 학파 기반으로 다시 설계했습니다.
- 중복 효과를 반복 사용하던 구성을 줄이고, 각 룬워드가 더 명확한 역할을 갖도록 조정했습니다.
- prefix 83개와 suffix 패밀리를 대규모로 재작업해 파밍 결과가 더 읽히고, utility prefix 3종도 전투용 효과로 복귀했습니다.
- `ConvertDamage`는 단일 우승 속성만 남기지 않고 원소별로 함께 적용되도록 고쳐, 체감과 설명이 더 일치하게 됐습니다.

### UI/UX

- 룬워드 워크벤치 패널은 레시피 프리뷰 기본 오픈, 스크롤 복구, 레이아웃 밀도 조정, 패널 이동/배치 개선을 거쳤습니다.
- Prisma control panel도 드래그/배치가 더 부드럽게 느껴지도록 후반 RC에서 정리했습니다.
- 어픽스 설명과 공개 문서를 대규모로 손봐, 한국어 설명이 더 구체적으로 보이고 플레이어용 문서에서 내부용 정의가 직접 드러나지 않게 정리했습니다.
- Bloom 계열 트랩은 `planted / burst` 피드백과 실패 진단이 추가되어 실제 발동 여부를 확인하기 쉬워졌습니다.

### 안정성/호환성

- 플레이어 피격 시 `TESHitEvent` fallback 재진입으로 이어지던 프리징을 수정했습니다.
- SE/AE 공용 CommonLibSSE 타깃 문제를 정리해 Address Library 초기화 실패를 막았습니다.
- 프로젝트 경로 이동 뒤 stale build tree와 릴리즈 검증이 깨지는 문제를 줄였습니다.
- generator/runtime contract sync, 패키징, Papyrus compile 경로도 후반 RC에서 안정화했습니다.

## RC 단계별 흐름

### rc02 ~ rc05: 룬워드와 어픽스 대수술 시작

- `rc02`: 룬워드 중복 효과 12종을 개별 고유 효과로 분리했습니다.
- `rc03`: 룬워드 actorValue 이름 정합성을 맞추고 composed spec을 다시 생성했습니다.
- `rc04`: 남아 있던 중복 룬워드 효과 19종을 다시 고유 효과로 정리했습니다.
- `rc05`: prefix 83개를 로어 기반으로 전면 리워크하고, suffix 8개 패밀리를 빌드 특화형으로 교체했습니다.

### rc06 ~ rc09: 데이터 검증과 공개 설명 정비

- `rc06`: `lint_affixes.py` actorValue 검증을 강화하고 `compose_affixes.py --sync-runtime`를 추가했습니다.
- `rc07`: 문서 디렉터리를 `docs/`로 통합하고, 방어구 suffix 드롭 풀 누락을 수정했습니다.
- `rc08`: 재변환 시 레시피 선택 UI가 갱신되지 않던 버그를 수정했습니다.
- `rc09`: 룬워드/프리픽스 설명을 더 구체적으로 정리하기 시작했습니다.

### rc10 ~ rc13: 설명 개선과 효과 계약 보정

- `rc10`: `ConvertDamage`를 Best Wins 방식에서 원소별 동시 적용 방식으로 변경했습니다.
- `rc11`: 243개 어픽스 설명을 한국어 서술형으로 전면 리라이트했습니다.
- `rc12`: utility prefix 3종 전투 효과를 재설계해 복귀시켰습니다.
- `rc12b`: 모듈 source의 `name/nameKo` 서술형을 compose 재생성 후에도 유지되도록 맞췄습니다.
- `rc13`: `AttackDamageMult`와 `WeaponSpeedMult` magnitude 단위가 100배로 부풀던 문제를 수정했습니다.

### rc14 ~ rc18: 내부 구조 정리와 워크벤치 UX 다듬기

- `rc14`: runtime ownership 추출과 affix proc 흐름 하드닝을 진행했고, runtime contract sync에 상대 경로 처리를 넣었습니다.
- `rc15`: Prisma/Calamity 패널 UI와 피드백을 다듬고 bundled bootstrap을 정리했습니다.
- `rc16`: 룬워드 recipe/workbench 레이아웃을 재조정했습니다.
- `rc17`: 룬워드 워크벤치 밀도를 조정하고 스크롤 레이아웃을 복구했습니다.
- `rc18`: runtime currency drop hook를 비활성화하고, 레시피 프리뷰를 기본 오픈 상태로 정리했습니다.

### rc19 ~ rc23: 릴리즈 안정화와 후반 핫픽스

- `rc19`: 경로 이동 뒤 stale build tree 자동 복구, 버전 단일 진실원, EventBridge/Prisma/serialization helper 분해를 반영했습니다.
- `rc20`: Prisma control panel 드래그 체감을 개선했습니다.
- `rc21`: SE/AE 공용 CommonLibSSE 타깃으로 되돌려 Address Library 초기화 문제를 수정했습니다.
- `rc22`: 플레이어 피격 프리징을 수정하고 incoming hit routing을 안정화했습니다.
- `rc23`: Bloom 트랩 피드백과 실패 진단, 특수 액션 계약 강화, debug HUD/log 분리, Linux/WSL 릴리즈 검증 보강을 반영했습니다.
