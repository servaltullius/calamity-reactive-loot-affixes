## Calamity Reactive Loot & Affixes v1.2.23

> [!WARNING]
> 이 버전은 QHD, FHD, 울트라와이드와 다양한 UI 환경에서 실사용 피드백을 받기 위한 공개 테스트(Pre-release)입니다. 설치 전 현재 저장 파일(`.ess`와 동명 `.skse`)을 백업하거나 별도 저장 슬롯을 사용해 주세요. 기존 `v1.2.21`은 안정 릴리스로 유지됩니다.

`v1.2.23`은 4K에서 확인된 어픽스 툴팁 외관은 그대로 유지하면서, 낮은 해상도에서는 화면 비율에 맞게 툴팁을 축소하는 UI 호환성 업데이트입니다. `v1.2.22`의 아이템 인스턴스 상태 안전성 수정과 `Reset State / 상태 초기화` 기능도 그대로 포함합니다.

### 해상도별 변화

| 해상도 | 툴팁 배율 | 최대 표시 폭 | 본문 유효 글자 크기 | 기본 위치 `right / top` |
|---|---:|---:|---:|---:|
| 4K `3840x2160` | `2.500` | `1800px` | `45px` | `70 / 255` |
| QHD `2560x1440` | `1.667` | `1200px` | `30px` | `47 / 170` |
| FHD `1920x1080` | `1.250` | `900px` | `22.5px` | `35 / 128` |
| UWQHD `3440x1440` | `1.667` | `1200px` | `30px` | `47 / 170` |

- 4K의 기존 `2.5x` 크기와 위치는 바뀌지 않습니다.
- QHD와 FHD에서는 화면 대비 툴팁 점유율이 4K와 비슷하도록 축소됩니다.
- 울트라와이드는 가로 폭이 아니라 더 작은 뷰포트 비율을 사용해 과도한 확대를 막습니다.
- 폭 `900px` 이하 분기와 기존 글자 크기 조절 `70%~180%`, `+/-8%` 버튼은 유지됩니다.

### 기존 설정과 호환성

- 세이브, 어픽스, 룬워드 런타임 저장 형식은 변경하지 않았습니다.
- 기존 `prisma_tooltip_layout.json`의 수동 위치와 글자 크기는 사용자의 배치를 갑자기 이동시키지 않도록 그대로 유지됩니다.
- 저장된 위치가 있는 상태에서 해상도를 바꾸면 크기는 자동으로 조절되지만 `right/top` 위치는 고정 픽셀로 유지됩니다.
- 현재 해상도에 맞는 새 기본 위치를 사용하려면 게임을 해당 해상도로 실행한 뒤 툴팁 UI의 `Reset`을 한 번 실행해 주세요.

### 검증

- 해상도별 자동 배율·기본 위치 계산과 저장 경로 회귀 테스트
- Python tools 테스트와 affix compose/lint
- .NET generator 테스트
- Linux clang-cl SKSE DLL 빌드
- 정적 정책 검사 및 runtime-gate
- MO2 ZIP 구조 및 SHA256 검증

실제 QHD/FHD/UWQHD 인게임 화면에서의 크기 체감은 자동 검사 범위가 아니며, 이번 공개 테스트에서 피드백을 수집합니다.

### 공개 테스트에서 받고 싶은 피드백

- 사용 해상도와 전체 화면/창 모드
- UI 배율 또는 UI 관련 모드 사용 여부
- 툴팁이 너무 크거나 작게 보이는 화면 캡처
- 기존 툴팁 배치를 저장한 적이 있다면 Reset 전후 차이
- 판매·상자 이동 뒤 다른 아이템으로 어픽스 상태가 이어지는 증상의 재발 여부
