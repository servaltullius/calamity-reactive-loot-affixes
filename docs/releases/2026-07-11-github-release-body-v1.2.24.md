## Calamity Reactive Loot & Affixes v1.2.24

무기 거치대의 `DropObject` 경로에서 아이템 인스턴스 상태가 다른 장비로 넘어갈 수 있던 문제를 막는 안전성 핫픽스입니다.

### 핵심 수정

- 어픽스 장비를 월드에 드롭하면 상태를 플레이어 UID에서 실제 월드 참조로 분리합니다.
- 같은 월드 참조를 다시 주울 때에만 기존 어픽스·룬워드·런타임 상태를 새 플레이어 UID로 복원합니다.
- 거치대에 둔 A와 같은 UID를 다른 창고의 B가 재사용해도 A의 상태가 B에 붙지 않습니다.
- 참조 핸들 재사용, 중복 이벤트, 목적지 상태 충돌은 병합하지 않고 안전하게 차단합니다.
- 저장·로드 후에도 동일한 거치대 무기를 회수할 수 있으며, 삭제된 월드 참조의 상태 정리를 보강했습니다.
- 구버전 세이브에 남은 orphan player-key를 로드 및 모드 재활성화 시 정리합니다.

### 기존 세이브 주의사항

- 아직 다른 아이템이 오염되지 않았다면, `v1.2.24` 로드 시 구버전의 남은 플레이어 UID 상태를 폐기해 추가 이전을 막습니다.
- 이미 다른 아이템에 잘못된 어픽스가 나타난 경우 자동으로 원래 아이템을 판별할 수 없습니다. 룬워드 패널의 `Reset State / 상태 초기화`를 사용하거나 문제 발생 전 세이브에서 다시 시작해 주세요.
- 구버전에서 이미 거치대에 올려둔 원본 장비는 소유권 증거가 남아 있지 않아 기존 어픽스가 보존되지 않을 수 있습니다.

### 호환성

- 세이브 직렬화 형식은 변경하지 않았습니다.
- Skyrim SE/AE용 단일 NG DLL 구성은 유지됩니다.
- `v1.2.23`의 4K/QHD/FHD/울트라와이드 툴팁 배율 수정은 그대로 포함됩니다.

### 권장 확인 순서

1. 어픽스 무기 A를 장착하고 무기 거치대에 겁니다.
2. 다른 창고에서 무기 B를 꺼내 B에 A의 어픽스가 나타나지 않는지 확인합니다.
3. 거치대의 A를 다시 회수해 A의 기존 어픽스가 유지되는지 확인합니다.
4. A를 다시 거치대에 둔 채 저장·로드한 후 2~3단계를 반복합니다.
