# Calamity - Reactive Loot & Affixes v1.3.1

> Pre-release / Prisma UI 안정성·복구 피드백용
> 날짜: 2026-07-13

## 요약

v1.3.1은 v1.3.0의 어픽스·룬워드 효과와 밸런스를 그대로 유지하면서, Prisma UI의 API 탐색, View 생성, DOM-ready, timeout과 재시도 수명주기를 안정화한 공개 테스트 패치입니다.

Prisma UI는 전체 툴팁과 룬워드·재련 패널을 위한 기본 프레임워크로 계속 사용합니다. Prisma UI를 사용할 수 없는 환경에서는 전체 UI를 흉내 내는 대체 화면 대신 MCM에서 현재 상태를 확인하고, 필요한 경우 세이브당 한 번 누락 통화를 안전하게 복구할 수 있습니다.

## Prisma UI 수명주기 안정화

- Prisma API 또는 View 생성 실패 시 5초 간격으로 다시 시도합니다.
- `CreateView()`가 유효하지 않은 handle을 반환하면 해당 View API를 호출하지 않습니다.
- View가 생성된 뒤 15초 안에 DOM-ready 콜백이 오지 않으면 해당 View를 폐기하고 재생성 경로로 전환합니다.
- 이전 View에서 늦게 도착한 DOM-ready 콜백은 현재 View 상태를 덮어쓰지 못합니다.
- 콜백과 활성 View 상태의 동시 접근, View 생성 중 동기 콜백이 도착하는 경우를 안전하게 처리합니다.
- 새 View가 준비되면 툴팁, 언어, 레이아웃, 룬워드 목록, 레시피와 패널 데이터를 다시 동기화합니다.
- 로딩 중 패널 열기 요청은 보존하지만, 실제 DOM-ready 전에는 패널이 열렸다는 성공 메시지를 표시하지 않습니다.
- MCM의 상태 확인은 정상 작동 중인 View를 파괴하거나 강제로 재생성하지 않는 비파괴 검사입니다.

## MCM 최소 안전 폴백

Prisma UI 페이지의 `Status & Safe Recovery / 상태·안전 복구` 구역에 다음 기능을 추가했습니다.

- **Prisma UI 상태 확인**: API 미탐지, View 로딩, 준비 완료, View 생성 실패, DOM-ready timeout 상태를 HUD에 표시합니다.
- **룬워드 상태 표시**: 선택한 베이스, 레시피 진행도와 필요한 룬 상태를 HUD와 로그에 표시합니다.
- **누락 통화 복구**: 세이브당 한 번 재련 오브 5개를 지급하고, 플레이어가 현재 하나도 보유하지 않은 룬 종류에만 파편 1개를 지급합니다.

통화 복구는 장비, 어픽스, 룬워드 선택 상태나 완성된 룬워드를 초기화하지 않습니다. 같은 세이브에서 두 번째 복구는 차단되며 이 상태는 저장·불러오기에 유지됩니다.

## Prisma UI를 사용할 수 없을 때

- 전리품, 어픽스, 룬 파편, 재련과 룬워드 게임플레이 로직은 계속 동작합니다.
- 전체 아이템 툴팁과 Prisma 룬워드·재련 패널은 사용할 수 없습니다.
- 패널 열기 요청은 실패 상태를 한 번 알리고 거짓 성공 메시지를 표시하지 않습니다.
- MCM Helper를 사용하는 환경에서는 위 상태 확인, 룬워드 상태 표시와 안전 복구 기능을 사용할 수 있습니다.

## 호환성 및 의존성

- 새 게임은 필요하지 않습니다.
- 세이브 및 어픽스·룬워드 직렬화 형식은 변경하지 않았습니다.
- 기존 장비, 보유 파편, 재련 오브와 완성된 룬워드는 그대로 유지됩니다.
- 어픽스·룬워드 효과 정의, 룬 분포, 기본 파편 드롭률 8%와 99회 무드롭 피티는 변경하지 않았습니다.
- SKSE64 및 Address Library 요구 사항은 기존과 같습니다.
- Prisma UI는 전체 툴팁과 조작 패널에 계속 필요합니다. 이번 MCM 기능은 전체 Prisma UI의 대체 구현이 아닙니다.

## 알려진 제한

- MCM 폴백은 상태 확인과 안전 복구만 제공합니다. 아이템별 툴팁, 레시피 선택과 재련 조작은 Prisma UI가 필요합니다.
- 누락 통화 복구는 과거에 실제로 잃어버린 수량을 추적하지 않습니다. 복구 시점에 보유량이 0인 룬 종류만 보충합니다.
- 자동 검증은 실제 게임 세션의 UI 콜백 순서와 모드 조합을 완전히 재현하지 않습니다.
