# Calamity - Reactive Loot & Affixes v1.4.0

> Pre-release / Corpse-only currency runtime, effect contract, and Prisma UI integrity update
> 날짜: 2026-07-14

## 요약

v1.4.0은 룬워드 조각과 재련 오브의 획득 경로를 전면 정리하는 공개 테스트 버전입니다.

이제 통화 판정은 **플레이어 또는 player-owned summon/proxy가 처치한 적대 대상의 death event**에서만 수행됩니다. 성공한 조각과 오브는 바닥에 생성되거나 플레이어에게 즉시 지급되지 않고, 사망한 적의 시체 인벤토리에 기존 전리품처럼 조용히 추가됩니다.

일반 상자·보스 상자·컨테이너, 시체 활성화, 아이템 픽업, 월드 생성과 새 SPID 통화 분배는 모두 통화 획득 경로에서 제외했습니다. 전투와 무관한 상자에서 통화가 나오거나 아이템이 갑자기 월드에 튀어나오던 부자연스러운 동작을 제거했습니다.

이번 버전에는 시체별 중복 방지와 피티 저장을 위한 `CCRT v1` 코세이브 레코드, Dragon 효과의 MGEF 분리, 룬워드·MGEF 생성 계약 강화, Prisma 레시피 상세 및 접근성 개선, fail-hard 릴리스 검증도 포함됩니다.

## 주요 변경 사항

### 적격 적대 시체 전용 통화 판정

룬워드 조각과 재련 오브의 실제 판정 권한을 SKSE `TESDeathEvent` 기반 **eligible hostile corpse-only** 경로로 일원화했습니다.

판정 대상:

- 플레이어가 직접 처치한 적대 대상
- 플레이어가 발사한 투사체나 마법으로 처치한 적대 대상
- player-owned summon/proxy가 처치한 적대 대상
- player-owned summon/proxy가 발사한 투사체로 처치한 적대 대상

판정 제외:

- 일반 상자와 보스 상자
- 모든 일반 컨테이너 활성화
- 시체를 열거나 반복 활성화하는 행동
- 아이템 픽업과 인벤토리 이동
- 월드 `PlaceAtMe` 및 바닥 생성
- 플레이어 인벤토리 직접 지급
- 새 SPID `Item`, `DeathItem`, Perk 통화 분배
- 독립 NPC 또는 일반 팔로워가 낸 처치
- 함정, Hazard, Explosion 등 환경 오브젝트가 낸 처치
- 비적대 대상
- 플레이어 동료·팔로워, player-owned 대상
- 소환·지휘 상태인 피해자
- 아동 대상

성공한 통화는 사망한 적의 인벤토리에 직접 삽입하며, 실제 수량이 증가한 경우에만 성공으로 확정합니다. 월드 생성이나 플레이어 직접 지급 fallback은 없습니다.

설정의 `loot.currencyDropMode=hybrid` 값은 기존 설정 파일과 MCM 계약을 깨지 않기 위한 **레거시 호환 토큰**으로만 남습니다. 실제 동작은 corpse-only입니다.

### 피티와 시체 중복 방지 저장

새로운 추가 코세이브 레코드 `CCRT v1`을 도입했습니다.

`CCRT`가 저장하는 상태:

- 룬워드 조각 실패 streak
- 재련 오브 실패 streak
- 처리한 시체 FormID
- 처리한 게임 날짜
- 룬 조각·재련 오브 카테고리별 processed mask

다음 중복 경로를 차단합니다.

- 동일 시체에 death event가 반복 전달되는 경우
- 저장 후 다시 불러왔을 때 동일 시체 이벤트가 다시 들어오는 경우
- 구 SPID 통화와 새 런타임 판정이 같은 시체에 겹치는 경우
- 룬 조각 또는 재련 오브 중 한 카테고리만 이미 존재하는 경우

부활 이벤트가 확인되면 해당 시체 ledger를 제거하므로, 부활한 대상의 다음 생명 주기에서는 다시 정상 판정할 수 있습니다.

확률이 `0%`이거나 실제 인벤토리 삽입이 실패했을 때는 기존 피티를 초기화하지 않습니다. 피티는 실제 아이템 추가 성공 때만 초기화됩니다.

### 기본 확률과 룬 분포 유지

이번 버전은 v1.2.25에서 조정한 룬 경제 분포를 다시 변경하지 않습니다.

- 룬워드 조각 기본 판정: `8%`
- 재련 오브 기본 판정: `5%`
- 룬 조각 99회 연속 실패 피티 유지
- `El-Amn = 4`
- `Sol-Um = 3`
- `Mal-Lo = 2`
- `Sur-Zod = 1`
- 최고·최저 가중치 비율: `4:1`

MCM 확률 변경은 다음 적격 적대 사망 판정부터 적용됩니다. 일반 상자, 픽업 또는 월드에는 적용할 통화 판정 자체가 없습니다.

### Dragon 효과와 생성 계약 보강

Dragon의 화염·냉기·전격 저항 효과가 Chains of Honor와 동일한 Magic Effect 레코드를 공유하던 구조를 분리했습니다.

- Dragon 전용 Fire/Frost/Shock MGEF 3개 추가
- 신규 MGEF는 기존 생성 레코드 뒤 append-only로 추가
- 기존 앞쪽 730개 생성 레코드의 할당 순서 유지
- 전체 생성 레코드 수: 733개
- 기존 어픽스 및 룬워드 ID 유지

생성기 검증도 강화했습니다.

- 정확히 94개 룬워드 레시피인지 검증
- 정확히 33개 룬 가중치인지 검증
- 레시피 ID와 결과 어픽스 ID의 대소문자 무시 중복 검사
- 모든 룬 가중치가 유한한 양수인지 검사
- 존재하지 않는 룬이나 결과 어픽스 참조 차단
- 사용되지 않는 룬 가중치 차단
- 동일 MGEF가 상충하는 cast type 또는 target type으로 생성되는 경우 즉시 실패
- 레거시 룬워드 fallback은 명시적인 복구 환경 변수 없이 사용할 수 없도록 제한

이번 변경은 전체 효과를 다시 밸런싱한 버전은 아닙니다. 기존 어픽스와 룬워드의 정체성 및 효과 구성은 유지하면서 Dragon 효과의 레코드 독립성과 생성 계약의 안전성을 보강했습니다.

### Prisma 레시피 탐색기 개선

- 효과, 권장 베이스, 룬 순서, 상세 설명의 한국어·영어 구조화 payload
- 선택 상태, 검색 결과, hover에서 동일한 상세 설명 사용
- 선택 직후 미리보기 즉시 갱신
- 장비 기본 어픽스와 레시피 설명을 함께 표시
- 패널을 실제 viewport 경계 안에 자동 배치
- 720p 및 글자 크기 180%에서 장문 툴팁 내부 스크롤 지원
- 내용이 실제로 바뀔 때만 툴팁 스크롤 위치 초기화
- 마우스 휠, scrollbar thumb, track click과 키보드 탐색 지원
- listbox·option·`aria-selected`·roving tabindex 보강
- 방향키, Home, End 탐색 지원
- reduced-motion 환경에서 불필요한 애니메이션 제거

v1.3.3에서 확인한 부드러운 wheel·scrollbar 동작은 유지합니다.

### 릴리스 패키징 검증 강화

릴리스 도구가 오래된 산출물을 조용히 재사용하지 않도록 fail-hard 방식으로 변경했습니다.

이제 다음 경우 패키징 또는 검증이 실패합니다.

- 방금 빌드한 SKSE DLL이 없는 경우
- 기존 `Data` DLL을 fresh build 대신 사용하려는 경우
- Papyrus 컴파일이 실패했는데 기존 PEX를 재사용하려는 경우
- 필수 PEX 3개가 정확히 일치하지 않는 경우
- ZIP 내부 `runtime_contract.json`이 현재 계약과 다른 경우
- ZIP 파일명 버전이 CMake 버전과 다른 경우
- ZIP CRC에 문제가 있는 경우
- 빌드 DLL과 ZIP 내부 DLL 해시가 다른 경우
- 필수 MO2 파일 구조가 어긋나는 경우

## 호환성

- 새 게임은 필요하지 않습니다.
- 기존 장비와 인스턴스 어픽스 상태를 유지합니다.
- 기존 보유 룬 조각과 재련 오브를 유지합니다.
- 기존 완성 룬워드와 재련 시 룬워드 보존 규칙을 유지합니다.
- 기존 어픽스·룬워드 ID를 유지합니다.
- 기존 `IAXF`, `RWRD`, `LCLD` 등 코세이브 레코드는 변경하지 않습니다.
- `CCRT v1`은 기존 형식을 교체하지 않는 추가 레코드입니다.
- 기본 조각 8%, 오브 5%, 4/3/2/1 룬 가중치와 99회 룬 조각 피티를 유지합니다.
- SPID는 현재 통화 드랍에 필요하지 않습니다.
- 동봉된 `CalamityAffixes_DISTR.ini`는 새 통화 분배 규칙이 없는 빈 호환 산출물입니다.

## MO2 업그레이드 주의

기존 버전을 그대로 둔 채 v1.4.0을 별도 MO2 모드로 겹쳐 활성화하지 않는 것을 권장합니다.

모드 충돌 우선순위나 수동 병합 결과에 따라 구 버전의 `CalamityAffixes_DISTR.ini`가 v1.4.0의 빈 호환 파일보다 우선하면 이전 SPID 통화 분배가 계속 작동할 수 있습니다.

안전한 업데이트 방법:

1. 기존 Calamity 모드를 v1.4.0으로 교체하거나 덮어씁니다.
2. 또는 기존 버전 모드를 비활성화합니다.
3. 별도 레이어로 설치해야 한다면 구 `CalamityAffixes_DISTR.ini`를 명시적으로 비활성화합니다.
4. 다른 모드가 별도의 SPID 통화 규칙을 배포하는지도 확인합니다.

SPID 자체가 설치되어 있는 것은 문제가 아닙니다. 중요한 것은 구 Calamity DISTR 규칙이 MO2 충돌에서 승리하지 않게 하는 것입니다.

## 알려진 전환 한계

업데이트 전에 구 SPID가 이미 통화 또는 해당 레벨드 리스트를 액터에게 넣었다면, v1.4.0은 이를 룬 조각과 재련 오브 카테고리별로 확인하고 같은 카테고리의 새 런타임 판정을 생략합니다.

다만 구 SPID 판정 결과가 “무드롭”이었던 액터에는 과거 결과를 증명할 아이템이나 마커가 남지 않을 수 있습니다. 그런 전환 액터는 v1.4.0에서 처음 죽을 때 새 corpse-only 판정을 한 번 받을 수 있습니다. 과거 무드롭 결과를 사후에 식별할 데이터가 없기 때문에 이 경우까지 완전히 구분할 수는 없습니다.

의도된 정책:

- 일반 팔로워가 단독으로 처치한 적은 통화 판정 대상이 아닙니다.
- 독립 NPC가 처치한 적도 대상이 아닙니다.
- 함정, 추락, 환경 오브젝트로 죽은 대상도 대상이 아닙니다.
- 일반 상자와 보스 상자는 통화 획득처가 아닙니다.

## 공개 테스트 요청

가능하면 다음 항목을 구분해 피드백해 주세요.

1. 플레이어 근접·활·마법 처치 후 통화가 해당 시체에만 들어가는지
2. player-owned summon/proxy의 처치가 정상적으로 플레이어 측 처치로 인정되는지
3. 일반 팔로워 또는 독립 NPC의 처치에서는 통화가 생기지 않는지
4. 함정, 추락, 환경 피해 처치에서는 통화가 생기지 않는지
5. 일반 상자와 보스 상자에서 통화가 나오지 않는지
6. 시체를 반복해서 열거나 저장·로드해도 중복 판정이 없는지
7. 부활한 대상을 다시 처치했을 때 새로운 생명 주기로 정상 판정되는지
8. 구 버전 세이브와 기존 장비·룬워드가 정상적으로 유지되는지
9. 레시피 탐색기의 wheel·scrollbar drag·키보드 스크롤이 해상도와 UI 배율에 관계없이 부드러운지
10. 레시피 효과·권장 베이스·룬 순서·상세가 한국어/영어에서 정확하고 읽기 쉬운지

룬 분포 피드백을 주실 때는 가능하면 다음 정보도 함께 부탁드립니다.

- 업데이트 후 획득한 룬 조각 50~100개의 룬별 개수
- 플레이 시간
- 직접 처치와 소환체 처치 비중
- 저급 레시피 첫 완성까지의 체감
- 고급 룬 및 고급 룬워드 접근성 체감

문제가 발생하면 다음 로그와 환경 정보를 함께 첨부해 주세요.

- `Documents/My Games/Skyrim Special Edition/SKSE/CalamityAffixes.log`
- 사용한 Skyrim 런타임 버전
- SKSE64, Address Library, Prisma UI 버전
- MO2 왼쪽 창에서 Calamity 모드 충돌 우선순위
- 구 `CalamityAffixes_DISTR.ini`가 남아 있는지 여부
